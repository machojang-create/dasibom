# 자동배포 연결 — Macho가 할 2가지 (웹 클릭만, 5분)

> 끝나면: 앞으로 코드가 GitHub에 올라가면 **자동으로 사이트에 배포**됩니다. (터미널 영영 안 씀)
> 워크플로 파일은 이미 제가 올려놨고, 아래 "키 발급 → GitHub에 등록"만 하시면 연결 완료.

---

## 작업 1. Firebase에서 "키" 발급
1. 이 링크 클릭(다시봄 만든 구글계정 machojang@gmail.com 로그인 상태):
   **https://console.firebase.google.com/project/mylife-650f0/settings/serviceaccounts/adminsdk**
2. 화면 아래쪽 **"새 비공개 키 생성 (Generate new private key)"** 파란 버튼 클릭
3. 팝업에서 다시 **"키 생성 (Generate key)"** 클릭 → **JSON 파일이 다운로드됨**
   (이름 예: `mylife-650f0-firebase-adminsdk-xxxxx.json`)

> ⚠️ 이 파일은 **마스터 열쇠(비밀)** 입니다.
> - 프로젝트 폴더에 두지 마세요. 이메일·메신저로 보내지 마세요. **저(클로드)에게도 보내지 마세요.**
> - 아래 GitHub 비밀칸에 붙여넣은 뒤에는 다운로드한 파일을 **삭제**해도 됩니다.

## 작업 2. GitHub에 그 키를 "비밀값"으로 등록
1. 이 링크 클릭(GitHub 로그인 상태):
   **https://github.com/machojang-create/dasibom/settings/secrets/actions/new**
2. **Name(이름)** 칸에 정확히 입력:  `FIREBASE_SERVICE_ACCOUNT`
   (대문자, 밑줄 _ , 오타 주의 — 한 글자라도 다르면 작동 안 함)
3. **Secret(값)** 칸:
   - 다운로드한 JSON 파일을 **메모장**으로 열기 (파일 우클릭 → 연결 프로그램 → 메모장)
   - **Ctrl+A**(전체선택) → **Ctrl+C**(복사)
   - GitHub의 Secret 칸 클릭 → **Ctrl+V**(붙여넣기) — `{` 로 시작해서 `}` 로 끝나는 긴 내용 전체
4. **"Add secret"** 버튼 클릭 → 끝!

---

## 연결 확인
- GitHub Actions 현황: **https://github.com/machojang-create/dasibom/actions**
- 비밀값 등록 직전에 돌아간 빨간 X(실패)는 **무시**하세요 — 키가 없어서 그런 것(정상).
- 키 등록 후: 제가 코드를 한 번 올리거나, Actions 탭에서 **"Run workflow"** 누르면 → **초록 체크 = 배포 완료.**
- 배포된 사이트: **https://mylife-650f0.web.app**
