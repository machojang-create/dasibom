## 백로그 (우선순위순)
- ☑ 1. **공개서재(library.html) 정합 점검·수리** — 자서전 v3 체계(questions_v3) 전환 후 서재가 정상 표시되는지 점검, 특히 "나만 보기(qPrivate)" 답변이 공개서재에 노출되지 않도록 제외 처리(미구현이면 구현). memoir_v8의 saveToFirestore가 저장하는 구조와 library의 읽기 구조 대조.
- ☑ 2. **홈에 마음의 유언 노출** — index.html에 /letter 진입점 추가(자서전 카드 근처 작은 링크 또는 카드, "나의 인생 전집 예정" 톤). 기존 레이아웃 손상 금지.
- ☑ 3. **B2G 월간 보고서 골격** — admin.html 안부 탭에 "보고서" 보기 추가: 월별 출석률·신호등 분포·조치기록 목록을 인쇄 친화(A4) 레이아웃으로. 더미+실데이터 겸용(기존 careRows 재사용).
- ☑ 4. **이용약관·개인정보처리방침 페이지** — terms.html / privacy.html 골격(시니어 가독, 다시봄 톤, [회사명]·[연락처] 플레이스홀더), footer 링크 연결, firebase.json rewrite 추가.
- ☑ 5. **PWA 설치성** — manifest.json(이름·아이콘·테마색)·아이콘 점검/추가, index에 link 태그. "홈 화면에 추가" 동작 확인 가능한 수준까지.
- ☑ 6. **인물 지도 UI 골격** — people.html: 수동 입력(이름·관계·메모)으로 인물 카드 목록, localStorage 보관, "전집 패키지 기능(준비 중)" 배너. AI 자동추출은 연결부만 주석 설계.
- ☑ 7. **관리자 폴리시** — 회원 상세 패널에 careLogs 최근 3건 표시. (2026-06-15: 안부대상 careBox에 유형 아이콘·날짜·메모 3건+총건수, 없으면 안내문구)
